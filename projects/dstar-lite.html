Dynamic Obstacle Avoidance with D* Lite (ROS 2 Nav2)

I implemented a D* Lite global planner as a ROS 2 Navigation2 plugin to handle dynamic environments. The planner incrementally repairs paths as the costmap changes, so the robot can keep moving while obstacles appear, disappear, or shift. This avoids full re-plans and keeps latency low.

Planner Design

  • Nav2 Plugin: clean interface with lifecycle nodes and standard planner API.
  • Incremental Updates: D* Lite maintains state keys and reuses previous search work.
  • Priority Queue: custom cell-based queue for fast key updates and lookups.
  • Dynamic Costmap: subscribes to layered costmap updates (inflation/obstacles) in real time.
  • Smoothing: simple post-processing for curvature and collision margin improvements.

Scenarios & Testing

  • Sudden obstacles: object drops onto the path → planner repairs route around it.
  • Occluded obstacles: obstacle revealed late by sensors → incremental update avoids last-second stops.
  • Close-proximity obstacles: tight spaces with changing gaps → stable re-planning without oscillation.

Results

  • Consistent real-time re-planning under changing costmaps without full restarts.
  • Smoother, feasible paths after light post-processing.
  • Cleaner Nav2 integration (configurable via YAML; works with standard costmap layers).

Stack

ROS 2 · Navigation2 · D* Lite · Dynamic Costmaps · Gazebo · RViz · TurtleBot3
